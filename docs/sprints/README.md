# Documentação de sprints

Cada sprint do projeto **Inteligência Auto Postos** é documentada com a mesma estrutura.

## Siglas

| Sigla | Significado |
|-------|-------------|
| **PDR** | Documento de Requisitos do Produto — problema, objetivo, usuários, escopo e resultados esperados |
| **RDC** | Regras de Domínio e Critérios — restrições, comportamentos obrigatórios e critérios de aceite de negócio/técnicos |

## Estrutura obrigatória

1. Identificação da sprint
2. PDR — Documento de Requisitos do Produto
3. Escopo e fora do escopo
4. Personas e permissões
5. RDC — Regras de domínio e critérios
6. Requisitos funcionais
7. Requisitos não funcionais
8. Casos de uso
9. Fluxos de UX
10. Arquitetura e decisões técnicas
11. Variáveis de ambiente
12. Modelo de dados
13. Contratos de API
14. Tratamento de erros
15. Critérios de aceite
16. Plano de testes
17. Definition of Ready
18. Definition of Done
19. Entregáveis
20. Tarefas técnicas sugeridas
21. Riscos e dependências
22. Status de implementação *(quando a sprint já foi executada)*
23. Script completo para o Cursor IA

## Regra de uso

- **Antes de codar:** preencher PDR, RDC, escopo e critérios de aceite.
- **Durante:** atualizar status de implementação e pendências.
- **Ao concluir:** marcar DoD, registrar gaps e gerar script da próxima sprint.
